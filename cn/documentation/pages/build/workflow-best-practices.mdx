## 设计原则

### 保持简单
从解决问题的最简单工作流开始。只在必要时添加复杂性。

### 模块化设计
将复杂的工作流分解为更小的、可重用的子工作流，或对重复逻辑使用迭代节点。

### 错误处理
始终为外部 API 调用和用户输入实现错误处理。

## 性能提示

<Tip>
  - 通过缓存结果最小化 API 调用
  - 使用条件节点跳过不必要的处理
  - 并行化独立操作
</Tip>
