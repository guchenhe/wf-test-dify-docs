有效的调试策略可以帮助您快速识别并解决 Dify 工作流中的问题。

## 常规调试策略

### 从简单开始
从最简单的工作流配置开始调试。逐步增加复杂性以隔离问题。

### 使用逐步执行
利用步骤运行功能逐个执行节点并观察中间结果。

<Tip>
  从调试工具栏启用步骤运行模式，在节点之间暂停执行。
</Tip>

## 常见调试模式

### 1. 变量检查
始终检查每一步的变量值，确保数据流正确传递。

```
Input → Process → Output
  ↓       ↓         ↓
Check   Check     Check
```

### 2. 错误处理
在关键节点实施适当的错误处理：

- 添加回退值
- 使用条件逻辑
- 设置错误通知

### 3. 日志策略
在工作流中战略性地添加日志节点以跟踪执行路径。

## 性能调试

### 识别瓶颈
- 检查日志中的节点执行时间
- 查找超时警告
- 监控 API 调用延迟

### 优化慢节点
- 减少数据载荷大小
- 在适当的地方使用缓存
- 并行化独立操作

## 生产环境测试

<Warning>
  在部署到生产环境之前，始终在预发布环境中测试更改。
</Warning>

### 安全测试实践
1. 创建用于测试的重复工作流
2. 使用镜像生产环境的测试数据
3. 部署后监控错误率
4. 准备好回滚计划
