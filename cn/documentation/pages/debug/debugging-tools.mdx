Dify 提供了一套全面的调试工具，帮助您构建可靠的工作流。

## 内置调试工具

### 调试控制台
调试控制台显示实时执行信息：
- 节点执行状态
- 变量值
- 错误消息
- 性能指标

### 断点
在特定节点设置断点以暂停执行：

1. 点击节点
2. 启用断点切换
3. 运行工作流
4. 执行在断点处暂停

<Info>
  断点仅在调试会话期间处于活动状态，不会影响生产运行。
</Info>

## 变量检查器

### 检查复杂数据
变量检查器支持：
- **嵌套对象**：展开和折叠对象属性
- **数组**：查看带索引的数组内容
- **长字符串**：截断显示并提供全文查看器
- **JSON 格式化**：语法高亮的 JSON 视图

### 变量监视
将变量添加到监视列表以跨多次运行跟踪其值：

```
监视列表：
├─ user_input (string)
├─ api_response (object)
└─ processing_result (array)
```

## 网络检查器

### API 调用监控
监控工作流进行的所有 API 调用：
- 请求 URL 和头部
- 请求/响应负载
- 响应时间
- 状态码

### 调试 API 问题
常见的 API 调试场景：
- **超时错误**：检查网络延迟
- **401/403 错误**：验证身份验证凭据
- **速率限制**：监控请求频率

## 性能分析器

### 执行时间线
可视化工作流执行时间线以识别：
- 长时间运行的节点
- 顺序与并行执行
- 节点间的等待时间

### 内存使用
跟踪内存消耗：
- 每个节点的内存使用情况
- 工作流总内存占用
- 内存泄漏检测

## 日志聚合

### 集中式日志记录
所有工作流日志聚合在单一视图中：
- 按严重性筛选（信息、警告、错误）
- 按关键字搜索
- 可导出用于外部分析

<Tip>
  使用一致格式的结构化日志记录，使日志分析更容易。
</Tip>

## 与外部工具的集成

### 导出调试数据
导出调试会话以在外部工具中进行分析：
- JSON 格式用于程序化访问
- CSV 格式用于电子表格分析
- 日志文件用于日志聚合平台
