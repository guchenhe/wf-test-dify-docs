你已经在 Dify 中构建了出色的应用。现在让我们将它交付给你的用户。每个 Dify 应用都会自动以多种方式提供服务——选择最适合你情况的方式。

## 从 Web 应用开始

通过 Web 应用分享是你最快的路径。这些应用在你创建任何应用时会自动生成，无需设置即可立即工作。

<Steps>
  <Step title="点击发布">
    在你的应用中点击"发布"以激活最新版本。
  </Step>
  <Step title="复制 URL">
    在发布部分找到你的 Web 应用链接。
  </Step>
  <Step title="立即分享">
    将链接发送给用户——他们可以立即开始使用你的应用。
  </Step>
</Steps>

<Tip>
Web 应用在任何设备上都能工作，并自动适应屏幕尺寸。无需应用商店审批或安装。
</Tip>

## 发布工作原理

当你发布应用时，Dify 会在所有渠道激活你的最新配置。以下是幕后发生的情况：

- **Web 应用**立即更新新功能和响应
- **API 端点**提供最新的模型和工作流配置
- **嵌入式组件**反映更新的样式和行为
- **MCP 服务器**提供对当前应用功能的访问

<Warning>
发布会用当前配置替换你的实时应用。用户将在下次交互中立即看到更改。
</Warning>

## 发布最佳实践

在分享你的应用之前，请确保你已配置这些设置：

- **应用描述** - 帮助用户了解你的应用功能
- **图标和品牌** - 使你的应用易于识别且专业
- **访问控制** - 决定你的应用应该是公开的还是需要身份验证
- **速率限制** - 保护你的应用免受过度使用（对于 API 访问尤其重要）

<Check>
所有发布方法都使用相同的应用配置。设置一次，随处发布。
</Check>
