工作区是您团队在 Dify 中的完整 AI 环境。它包含并隔离您组织所需的一切：应用程序、知识库、团队成员、模型配置、插件和计费。将其视为您团队专用的 AI 开发和部署环境。

## 工作区概念模型

Dify 中的每个资源都属于一个工作区。当您创建应用时，它会继承工作区的模型配置。当您添加团队成员时，他们会根据其角色获得对工作区资源的访问权限。当您配置模型或安装插件时，它们将对整个工作区可用。

```
🏢 您的组织
└── 📁 工作区
    ├── 🤖 应用（聊天机器人、工作流、智能体）
    ├── 📊 知识库（文档、嵌入向量）
    ├── 👥 团队成员（角色和权限） 
    ├── 🧠 模型提供商（API 密钥、配置）
    ├── 🔧 工具和插件（集成、自定义代码）
    └── 💳 计费（订阅、限制、使用量）
```

这种工作区优先的设计意味着您的资源与其他组织完全隔离，团队成员只能访问他们被允许查看的内容，并且您只需为整个工作区配置一次模型和计费。

## 工作区创建

**Dify Cloud** 在首次登录时自动创建工作区。您将成为拥有完整权限的所有者。

**社区版** 在安装期间创建一个工作区。管理员邮箱和密码在设置过程中设定。

当您需要在不同法律实体、监管环境或客户项目之间实现完全隔离时，支持**多个工作区**。大多数组织使用单个工作区。

<Info>
您的个人账户可以属于多个工作区。使用左上角的工作区选择器在它们之间切换。
</Info>

## 资源如何连接

您构建的应用程序可以使用工作区中配置的任何模型提供商，访问所有工作区知识库，并利用已安装的插件。团队成员根据其工作区权限查看应用程序。

工作区角色决定对所有资源的访问权限：
- **所有者** 控制计费、模型提供商和工作区设置
- **管理员** 管理团队成员并配置模型/插件  
- **编辑者** 构建应用程序并管理知识库
- **成员** 使用已发布的应用程序
- **数据集操作员** 专注于知识库管理

## 工作区导航

Dify 围绕工作区概念组织一切。主导航显示您工作区中可用的应用、知识和工具。设置包含工作区范围的配置：成员、模型提供商、插件、计费（仅限云端）和个人账户设置。
