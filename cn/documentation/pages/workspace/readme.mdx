工作空间是您团队在 Dify 中完整的 AI 环境。它包含并隔离您组织所需的一切：应用程序、知识库、团队成员、模型配置、插件和计费。将其视为您团队专用的 AI 开发和部署环境。

## 工作空间概念模型

Dify 中的每个资源都属于一个工作空间。当您创建应用程序时，它会继承工作空间的模型配置。当您添加团队成员时，他们会根据自己的角色获得对工作空间资源的访问权限。当您配置模型或安装插件时，它们对整个工作空间都可用。

```
🏢 您的组织
└── 📁 工作空间
    ├── 🤖 应用程序（聊天机器人、工作流、智能体）
    ├── 📊 知识库（文档、嵌入向量）
    ├── 👥 团队成员（角色和权限）
    ├── 🧠 模型提供商（API 密钥、配置）
    ├── 🔧 工具和插件（集成、自定义代码）
    └── 💳 计费（订阅、限制、使用情况）
```

这种工作空间优先的设计意味着您的资源与其他组织完全隔离，团队成员只能访问他们被允许查看的内容，您只需为整个工作空间配置一次模型和计费。

## 工作空间创建

**Dify Cloud** 在首次登录时自动创建工作空间。您将成为拥有完全权限的所有者。

**社区版** 在安装过程中创建一个工作空间。管理员邮箱和密码在设置期间配置。

当您需要在不同法律实体、监管环境或客户项目之间实现完全隔离时，支持**多个工作空间**。大多数组织使用单个工作空间。

<Info>
您的个人账户可以属于多个工作空间。使用左上角的工作空间选择器在它们之间切换。
</Info>

## 资源如何连接

您构建的应用程序可以使用工作空间中配置的任何模型提供商，访问所有工作空间知识库，并利用已安装的插件。团队成员根据其工作空间权限查看应用程序。

工作空间角色决定对所有资源的访问权限：
- **所有者**控制计费、模型提供商和工作空间设置
- **管理员**管理团队成员并配置模型/插件
- **编辑者**构建应用程序并管理知识库
- **成员**使用已发布的应用程序
- **数据集操作员**专注于知识库管理

## 工作空间导航

Dify 围绕工作空间概念组织一切。主导航显示工作空间中可用的应用程序、知识和工具。设置包含工作空间范围的配置：成员、模型提供商、插件、计费（仅云版）和个人账户设置。
