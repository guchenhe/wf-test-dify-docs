# 増分更新テスト

このファイルは、増分更新ワークフローをテストするために**2回目のコミット**として追加されました。

## 目的

すでに翻訳PRがあるPRに追加のコミットをプッシュした場合のテストです：

1. **Update Translation PR** ワークフローがトリガーされる（Executeではない）
2. システムが翻訳ブランチから**最後に処理されたコミット**を検出
3. 増分範囲を計算：`last_processed..new_head`
4. 最後の同期以降の**変更/追加されたファイル**のみを翻訳
5. 新しいコミットで翻訳PRを更新

## ワークフローの動作

### 最初のコミット（処理済み）
- 追加：`workflow-validation.mdx`
- 変更：`quick-start.mdx`
- 移動：`key-concepts`
- 結果：翻訳PR #252が作成された

### 2回目のコミット（このファイル）
- 追加：`incremental-update-test.mdx`
- トリガーされるべき：**Update Translation PR** ワークフロー
- 更新されるべき：PR #252に新しい翻訳が追加

## 検証ポイント

✅ 増分更新ワークフローがトリガーされる
✅ 最後に処理されたコミットを正しく検出
✅ 新規/変更されたファイルのみを翻訳
✅ 既存の翻訳PRを更新（新しいものを作成しない）
✅ 翻訳ブランチにコミットを追加
✅ 両方のPRに更新についてコメント
