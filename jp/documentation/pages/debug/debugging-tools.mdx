Difyは信頼性の高いワークフローを構築するのに役立つ包括的なデバッグツールスイートを提供しています。

## 内蔵デバッグツール

### デバッグコンソール
デバッグコンソールはリアルタイムの実行情報を表示します：
- ノードの実行状況
- 変数の値
- エラーメッセージ
- パフォーマンス指標

### ブレークポイント
特定のノードにブレークポイントを設定して実行を一時停止：

1. ノードをクリック
2. ブレークポイントトグルを有効にする
3. ワークフローを実行
4. ブレークポイントで実行が一時停止

<Info>
  ブレークポイントはデバッグセッション中のみアクティブで、本番環境の実行には影響しません。
</Info>

## 変数インスペクター

### 複合データの検査
変数インスペクターは以下をサポート：
- **ネストしたオブジェクト**: オブジェクトプロパティの展開と折りたたみ
- **配列**: インデックス付きで配列の内容を表示
- **大きな文字列**: 完全なテキストビューアーによる切り詰め表示
- **JSON形式**: シンタックスハイライトされたJSONビュー

### 変数ウォッチ
変数をウォッチリストに追加して、複数回の実行にわたって値を追跡：

```
Watch List:
├─ user_input (string)
├─ api_response (object)
└─ processing_result (array)
```

## ネットワークインスペクター

### API呼び出し監視
ワークフローによって行われるすべてのAPI呼び出しを監視：
- リクエストURLとヘッダー
- リクエスト/レスポンスペイロード
- レスポンス時間
- ステータスコード

### API問題のデバッグ
一般的なAPIデバッグシナリオ：
- **タイムアウトエラー**: ネットワーク遅延をチェック
- **401/403エラー**: 認証クレデンシャルを確認
- **レート制限**: リクエスト頻度を監視

## パフォーマンスプロファイラー

### 実行タイムライン
ワークフロー実行タイムラインを可視化して以下を特定：
- 長時間実行されるノード
- 順次実行 vs. 並列実行
- ノード間の待機時間

### メモリ使用量
メモリ消費量を追跡：
- ノード毎のメモリ使用量
- ワークフロー全体のメモリフットプリント
- メモリリーク検出

## ログ集約

### 集約ログ
すべてのワークフローログを単一のビューに集約：
- 重要度で絞り込み可能（Info、Warning、Error）
- キーワード検索可能
- 外部分析用にエクスポート可能

<Tip>
  ログ分析を簡単にするため、一貫した形式の構造化ログを使用してください。
</Tip>
