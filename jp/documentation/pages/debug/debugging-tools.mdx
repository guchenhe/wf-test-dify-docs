Difyは、信頼性の高いワークフローを構築するための包括的なデバッグツールスイートを提供します。

## 組み込みデバッグツール

### デバッグコンソール
デバッグコンソールはリアルタイムの実行情報を表示します：
- ノード実行ステータス
- 変数値
- エラーメッセージ
- パフォーマンス指標

### ブレークポイント
特定のノードにブレークポイントを設定して実行を一時停止します：

1. ノードをクリック
2. ブレークポイントトグルを有効にする
3. ワークフローを実行
4. ブレークポイントで実行が一時停止

<Info>
  ブレークポイントはデバッグセッション中のみアクティブで、本番実行には影響しません。
</Info>

## 変数インスペクター

### 複雑なデータの検査
変数インスペクターは以下をサポートします：
- **ネストされたオブジェクト**: オブジェクトプロパティの展開と折りたたみ
- **配列**: インデックス付きで配列の内容を表示
- **大きな文字列**: 全文ビューアーによる短縮表示
- **JSON形式**: シンタックスハイライト付きのJSONビュー

### 変数ウォッチ
変数をウォッチリストに追加して、複数の実行にわたってその値を追跡します：

```
ウォッチリスト:
├─ user_input (string)
├─ api_response (object)
└─ processing_result (array)
```

## ネットワークインスペクター

### API呼び出し監視
ワークフローによって行われるすべてのAPI呼び出しを監視します：
- リクエストURLとヘッダー
- リクエスト/レスポンスペイロード
- レスポンス時間
- ステータスコード

### APIの問題のデバッグ
一般的なAPIデバッグシナリオ：
- **タイムアウトエラー**: ネットワーク遅延を確認
- **401/403エラー**: 認証資格情報を検証
- **レート制限**: リクエスト頻度を監視

## パフォーマンスプロファイラー

### 実行タイムライン
ワークフロー実行タイムラインを視覚化して以下を特定します：
- 長時間実行されるノード
- 順次実行 vs. 並列実行
- ノード間の待機時間

### メモリ使用量
メモリ消費を追跡します：
- ノード別メモリ使用量
- 総ワークフローメモリフットプリント
- メモリリーク検出

## ログ集約

### 集約ログ
すべてのワークフローログが単一のビューに集約されます：
- 重要度でフィルタリング可能（情報、警告、エラー）
- キーワードで検索可能
- 外部分析用にエクスポート可能

<Tip>
  ログ分析を容易にするために、一貫した形式の構造化ログを使用してください。
</Tip>
