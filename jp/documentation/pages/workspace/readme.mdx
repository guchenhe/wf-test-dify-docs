ワークスペースは、Difyにおけるチームの完全なAI環境です。組織が必要とするすべてを含み、分離します：アプリケーション、ナレッジベース、チームメンバー、モデル設定、プラグイン、請求。チーム専用のAI開発・展開環境と考えてください。

## ワークスペースのメンタルモデル

Difyのすべてのリソースはワークスペースに属します。アプリを作成すると、ワークスペースのモデル設定を継承します。チームメンバーを追加すると、彼らは役割に基づいてワークスペースリソースにアクセスできます。モデルを設定したりプラグインをインストールすると、それらはワークスペース全体で利用可能になります。

```
🏢 あなたの組織
└── 📁 ワークスペース
    ├── 🤖 アプリ（チャットボット、ワークフロー、エージェント）
    ├── 📊 ナレッジベース（文書、埋め込み）
    ├── 👥 チームメンバー（役割と権限）
    ├── 🧠 モデルプロバイダー（APIキー、設定）
    ├── 🔧 ツールとプラグイン（統合、カスタムコード）
    └── 💳 請求（サブスクリプション、制限、使用量）
```

このワークスペース優先の設計により、リソースは他の組織から完全に分離され、チームメンバーは許可されたもののみにアクセスでき、モデルと請求をワークスペース全体に対して一度設定できます。

## ワークスペース作成

**Dify Cloud**は初回ログイン時に自動的にワークスペースを作成します。フル権限を持つオーナーになります。

**Community Edition**はインストール中に1つのワークスペースを作成します。管理者のメールアドレスとパスワードはセットアップ時に設定されます。

**複数のワークスペース**は、異なる法人、規制環境、または顧客プロジェクト間で完全な分離が必要な場合にサポートされます。ほとんどの組織は単一のワークスペースを使用します。

<Info>
個人アカウントは複数のワークスペースに属することができます。左上角のワークスペースセレクターを使用してそれらを切り替えます。
</Info>

## リソースの接続方法

構築するアプリケーションは、ワークスペースで設定されたすべてのモデルプロバイダーを使用し、すべてのワークスペースナレッジベースにアクセスし、インストールされたプラグインを活用できます。チームメンバーは、ワークスペース権限に基づいてアプリケーションを見ることができます。

ワークスペースの役割は、すべてのリソースへのアクセスを決定します：
- **オーナー**は請求、モデルプロバイダー、ワークスペース設定を制御
- **管理者**はチームメンバーを管理し、モデル/プラグインを設定
- **編集者**はアプリケーションを構築し、ナレッジベースを管理
- **メンバー**は公開されたアプリケーションを使用
- **データセットオペレーター**はナレッジベース管理に注力

## ワークスペースナビゲーション

Difyはワークスペースの概念を中心にすべてを整理します。メインナビゲーションには、ワークスペース、ナレッジ、ツールが表示されます。設定には、ワークスペース全体の設定が含まれます：メンバー、モデルプロバイダー、プラグイン、請求（Cloudのみ）、個人アカウント設定。
