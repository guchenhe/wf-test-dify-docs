ワークスペースは、Difyにおけるチームの完全なAI環境です。組織が必要とするすべてを含み、分離します：アプリケーション、ナレッジベース、チームメンバー、モデル設定、プラグイン、課金。チーム専用のAI開発・デプロイメント環境として考えてください。

## ワークスペースの概念モデル

Difyのすべてのリソースはワークスペースに属します。アプリを作成すると、ワークスペースのモデル設定を継承します。チームメンバーを追加すると、役割に基づいてワークスペースリソースへのアクセス権を取得します。モデルを設定したりプラグインをインストールしたりすると、それらはワークスペース全体で利用可能になります。

```
🏢 あなたの組織
└── 📁 ワークスペース
    ├── 🤖 アプリ（チャットボット、ワークフロー、エージェント）
    ├── 📊 ナレッジベース（ドキュメント、埋め込み）
    ├── 👥 チームメンバー（役割と権限） 
    ├── 🧠 モデルプロバイダー（APIキー、設定）
    ├── 🔧 ツール＆プラグイン（統合、カスタムコード）
    └── 💳 課金（サブスクリプション、制限、使用量）
```

このワークスペースファーストな設計により、あなたのリソースは他の組織から完全に分離され、チームメンバーは許可されたもののみにアクセスでき、ワークスペース全体に対してモデルと課金を一度設定するだけで済みます。

## ワークスペースの作成

**Dify Cloud**では、初回ログイン時に自動的にワークスペースが作成されます。あなたは完全な権限を持つオーナーになります。

**Community Edition**では、インストール中に1つのワークスペースが作成されます。管理者のメールアドレスとパスワードはセットアップ時に設定されます。

**複数のワークスペース**は、異なる法人、規制環境、またはクライアントプロジェクト間で完全な分離が必要な場合にサポートされます。ほとんどの組織は単一のワークスペースを使用します。

<Info>
個人アカウントは複数のワークスペースに属することができます。左上角のワークスペースセレクターを使用して切り替えできます。
</Info>

## リソースの接続方法

構築するアプリケーションは、ワークスペースで設定されたすべてのモデルプロバイダーを使用し、すべてのワークスペースナレッジベースにアクセスし、インストールされたプラグインを活用できます。チームメンバーは、ワークスペース権限に基づいてアプリケーションを参照できます。

ワークスペースの役割は、すべてのリソースへのアクセスを決定します：
- **オーナー**は課金、モデルプロバイダー、ワークスペース設定を制御
- **管理者**はチームメンバーを管理し、モデル/プラグインを設定  
- **編集者**はアプリケーションを構築し、ナレッジベースを管理
- **メンバー**は公開されたアプリケーションを使用
- **データセット運用者**はナレッジベース管理に特化

## ワークスペースのナビゲーションの概念を中心にすべてを整理しています。メインナビゲーションには、ワークスペースで利用可能なアプリ、ナレッジ、ツールが表示されます。設定には、ワークスペース全体の設定が含まれます：メンバー、モデルプロバイダー、プラグイン、課金（Cloudのみ）、および個人アカウント設定。
