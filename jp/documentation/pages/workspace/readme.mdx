ワークスペースは、Difyにおけるチームの完全なAI環境です。組織が必要とするすべてを含み、分離します：アプリケーション、ナレッジベース、チームメンバー、モデル設定、プラグイン、そして請求。チーム専用のAI開発・デプロイメント環境として考えてください。

## ワークスペースの概念モデル

Difyのすべてのリソースはワークスペースに属します。アプリを作成すると、ワークスペースのモデル設定を継承します。チームメンバーを追加すると、彼らは役割に基づいてワークスペースリソースへのアクセス権を得ます。モデルを設定したりプラグインをインストールすると、それらはワークスペース全体で利用可能になります。

```
🏢 あなたの組織
└── 📁 ワークスペース
    ├── 🤖 アプリ (チャットボット、ワークフロー、エージェント)
    ├── 📊 ナレッジベース (ドキュメント、エンベディング)
    ├── 👥 チームメンバー (役割と権限) 
    ├── 🧠 モデルプロバイダー (APIキー、設定)
    ├── 🔧 ツール&プラグイン (統合、カスタムコード)
    └── 💳 請求 (サブスクリプション、制限、使用量)
```

このワークスペース優先設計により、あなたのリソースは他の組織から完全に分離され、チームメンバーは許可されたもののみにアクセスでき、モデルと請求はワークスペース全体で一度設定するだけで済みます。

## ワークスペースの作成

**Dify Cloud**では、初回ログイン時に自動的にワークスペースが作成されます。あなたは完全な権限を持つオーナーになります。

**コミュニティエディション**では、インストール時に1つのワークスペースが作成されます。管理者のメールアドレスとパスワードは、セットアップ時に設定されます。

**複数のワークスペース**は、異なる法人、規制環境、またはクライアントプロジェクト間で完全な分離が必要な場合にサポートされます。ほとんどの組織は単一のワークスペースを使用します。

<Info>
個人アカウントは複数のワークスペースに属することができます。左上角のワークスペース選択機能を使用して切り替えができます。
</Info>

## リソースの接続方法

構築するアプリケーションは、ワークスペースで設定されたすべてのモデルプロバイダーを使用でき、すべてのワークスペースナレッジベースにアクセスし、インストールされたプラグインを利用できます。チームメンバーは、ワークスペースの権限に基づいてアプリケーションを閲覧します。

ワークスペースの役割は、すべてのリソースへのアクセスを決定します：
- **オーナー** 請求、モデルプロバイダー、ワークスペース設定を制御
- **管理者** チームメンバーを管理し、モデル/プラグインを設定  
- **編集者** アプリケーションを構築し、ナレッジベースを管理
- **メンバー** 公開されたアプリケーションを使用
- **データセットオペレーター** ナレッジベース管理に集中

## ワークスペースナビゲーション

Difyはすべてをワークスペースの概念を中心に整理しています。メインナビゲーションは利用可能なアプリ、ナレッジ、ツールを表示します。設定にはワークスペース全体の設定が含まれます：メンバー、モデルプロバイダー、プラグイン、請求（クラウドのみ）、および個人のアカウント設定。
